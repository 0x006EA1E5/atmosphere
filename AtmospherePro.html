Atmosphere Satellite

Application using Satellite gain the following superpowers:
  • Elastic Scalability (just add more servers and they cluster automagically)
  • Super Speeds (in memory transaction speeds)
  • High Availability
  • Fault Tolerance
  • Cloud Readiness
  • 100% State Replication
Satellite is easily able to handle this type of case with in-memory performance, linear scalability as you add new nodes and reliability. Satellite is a small, and gets transparently enabled by Atmosphere. Because of this approach as well as Satellite’s ability to automatically discover and cluster with peers, Satellite provides drop-in session clustering ability for any Atmosphere enabled server. It requires no additional investment in hardware and elastically scales as you add app servers. This is a great way to ensure that session Atmosphere's state is maintained when you are clustering Atmosphere servers.

Atmosphere Postman

With Guaranteed Delivery, the Atmosphere Postman system uses a built-in data store to persist messages. Each Atmosphere Satellite on which the messaging system is installed on has its own data store so that the messages can be stored locally. When a websocket or fallback transport delivers a message, the send operation does not complete successfully until the message is safely stored in the sender’s data store. Subsequently, the message is not deleted from one data store until it is successfully forwarded to and stored in the next data store. In this way, once a websocket or fallback transport successfully sends the message, it is always stored on memory on at least one Atmosphere Satellite until is successfully delivered to and acknowledged by the browser.
